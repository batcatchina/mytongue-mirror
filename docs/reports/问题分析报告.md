# 舌镜项目问题修复报告

## 问题1：病机分析没有出现

### 根本原因
Bot 返回的格式是 Markdown 格式（无粗体）：
```
病机分析：舌淡红，苔薄白，舌形正常，提示病位在脾胃...
```

但前端 `api.ts` 中的正则表达式要求的是带 Markdown 粗体的格式：
```
/\*\*病机分析\*\*[：:]\s*([^\n]+)/
```

这导致正则匹配失败，`pathogenesis` 被设置为空字符串。

### 修复内容
修改 `src/services/api.ts` 中的 `parseMarkdownDiagnosis` 函数：

1. **主要证型、证型得分、病机分析、刺法、治疗频次** - 同时支持粗体和非粗体格式
2. **病机分析** - 支持捕获多行内容并合并为单行（从"病机分析："到下一个空行或"针灸"章节）

```typescript
// 病机分析 - 同时支持 **病机分析** 和 病机分析 两种格式
const pathogenesisMultiMatch = markdown.match(/病机分析[：:]\s*([\s\S]*?)(?=\n\n|\n针灸|##|$)/);
const pathogenesisMatch = markdown.match(/\*\*病机分析\*\*[：:]\s*([^\n]+)/)
  || markdown.match(/病机分析[：:]\s*([^\n]+)/);
const pathogenesis = pathogenesisMultiMatch 
  ? pathogenesisMultiMatch[1].trim().replace(/\n+/g, ' ')
  : (pathogenesisMatch ? pathogenesisMatch[1].trim() : '');
```

### 测试结果
使用 Bot 实际返回的内容进行正则测试，结果如下：
```
主要证型: 脾胃气滞
证型得分: 6
病机分析: 舌淡红，苔薄白，舌形正常，提示病位在脾胃；患者胃脘不适、腹胀、食欲不振，为脾胃气机阻滞，运化功能减弱...
刺法: 中脘、足三里用平补平泻法；内关、公孙用泻法；太冲用泻法。可配...
治疗频次: 每周2-3次，8次为一疗程
主穴: 中脘、足三里、内关、公孙、太冲...
```

## 问题2：舌照上传问题

### API 测试结果
经过测试，Coze API 本身能够正确处理图片请求：
- API 调用成功
- Bot 能够接收和处理图片数据
- Bot 返回了完整的辨证结果

### 可能原因
1. **图片数据格式问题**：确保 base64 URL 格式正确
2. **图片数据太长**：Coze API 对请求体大小有限制
3. **网络问题**：请求超时

### 修复内容
1. 添加调试日志，输出图片数据长度
2. 确保 base64 URL 格式正确（添加 `data:image/xxx;base64,` 前缀）

```typescript
const imageUrl = input.imageData.startsWith('data:') 
  ? input.imageData 
  : `data:image/jpeg;base64,${input.imageData}`;

console.log(`[舌照上传] 图片数据长度: ${imageUrl.length} 字符`);
```
