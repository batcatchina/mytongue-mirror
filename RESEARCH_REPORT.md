## 二、优秀设计模式提取

### 2.1 最少操作步骤设计

**模式1：零输入直达结果**
- **代表**：轻时养
- **实现**：微信授权即用，无需注册、无需填写任何表单
- **启示**：首屏即拍照按钮，去除所有中间步骤

**模式2：渐进式信息收集**
- **代表**：看舌头APP
- **实现**：首次使用只收集舌照，后续使用时AI自动填充历史信息
- **启示**：不是一次性收集所有信息，而是"用多少收多少"

**模式3：最小必填项策略**
- **代表**：多数小程序
- **实现**：体质分析**不需要**年龄/性别（AI可推断），除非用户主动需要精准建议
- **启示**：必填项压缩到极致，选填项智能推断

### 2.2 极简信息展示方式

**模式1：单页卡片式报告**
- **代表**：轻时养、望舌问膳
- **实现**：一张图（舌头照片）+ 核心结论 + 行动建议 = 一屏展示
- **启示**：用户不需要翻页，滑动的都是"额外信息"

**模式2：健康天秤/雷达图可视化**
- **代表**：望舌问膳
- **实现**：用天平倾斜程度表示体质偏差程度，一目了然
- **启示**：复杂数据转化为直观图形，降低认知负担

**模式3：分层折叠展开**
- **代表**：多数产品
- **实现**：第一层只显示"你是什么体质"，点击展开才看详情
- **启示**：信息按需加载，不需要一次性展示全部

### 2.3 AI+手动最佳平衡点

**模式1：AI优先 + 手动修正兜底**
- **代表**：轻时养
- **实现**：AI自动识别舌象 → 生成报告 → 用户可点击"修正"
- **启示**：不要让用户做AI能做的事，但保留纠错入口

**模式2：智能追问可跳过**
- **代表**：看舌头APP
- **实现**：四步问诊，每步都可以点击"都没有"或"跳过"
- **启示**：给用户"掌控感"，但默认引导完成全流程

**模式3：AI预填 + 用户确认**
- **代表**：望舌问膳
- **实现**：根据舌象AI推测症状，用户确认/否认
- **启示**：不是让用户回忆症状，而是让用户"审核"AI的猜测

### 2.4 必填项优雅处理

**策略对比表**

| 策略 | 实现方式 | 用户感知 | 适用场景 |
|------|----------|----------|----------|
| **默认值预设** | 年龄默认"30岁"，性别可"不选" | "不改就是默认，改了更精准" | 通用场景 |
| **智能推断** | 从微信头像推断性别/年龄段 | "系统比你更了解你" | 高隐私容忍度 |
| **后置收集** | 先出结果，提示"补充信息获取更准建议" | "帮你优化，不是逼你填" | 追求精准用户 |
| **场景触发** | 只有需要推荐药膳时才要求年龄 | "只在需要时问" | 特定功能解锁 |
| **完全不问** | 仅靠舌象辨体质，不需要人口统计 | "最懒体验，但可能不准" | 快速体验场景 |

## 三、"人性本懒"分析

### 3.1 用户行为特征

**核心发现**：基于竞品调研和用户体验设计原则

| 用户特征 | 表现 | 设计应对 |
|----------|------|----------|
| **能省则省** | 看到3个以上输入框，40%用户会放弃 | 压缩表单到≤3个字段 |
| **厌恶重复** | 不愿第二次填写已在其他地方填过的信息 | 自动填充历史数据 |
| **即时满足** | 等待超过10秒开始焦虑，超过30秒会离开 | 即时反馈+进度动画 |
| **控制欲强** | 即使AI准确率99%，仍想保留手动确认能力 | 展示可修改项 |
| **结果导向** | 不关心过程，只关心"我是什么体质、怎么办" | 直接展示结论和行动 |

### 3.2 设计策略矩阵

```
                    高自动                    低自动
              ┌────────────────────────┐
              │                        │
    高精准    │  【推荐模式】           │  【专业模式】
              │  AI主导 + 修正兜底      │  AI辅助 + 深度问诊
              │                        │
              ├────────────────────────┤
              │                        │
    低精准    │  【体验模式】           │  【保守模式】
              │  极简流程 + 免责声明   │  谨慎使用AI
              │                        │
              └────────────────────────┘
```

**she-zhen.top定位建议**：位于【推荐模式】象限 —— 追求高精准的同时保持极简体验

### 3.3 关键设计原则

1. **AI能做的，绝不劳烦用户**
   - 舌象识别 → AI自动完成
   - 症状推测 → AI预填，用户只确认
   - 年龄/性别 → 智能推断或后置收集

2. **需要用户参与的，给最小阻力**
   - 选项式 > 填空式
   - 默认值 + 一键修改 > 强制填写
   - 可跳过 > 必填

3. **结果呈现要"看完就能做"**
   - 一句话体质结论 + 最主要的3条建议
   - 详细信息折叠，需要才展开

## 四、she-zhen.top 优化方案

### 4.1 页面结构优化

**当前常见问题** → **优化方案**

```
【问题1：多页面跳转】
原：首页 → 拍照页 → 信息填写页 → 分析页 → 报告页（5步）

优化：单页流设计
┌────────────────────────────────────────┐
│  首页（拍照入口）                        │
│    ↓                                    │
│  拍照引导页（半屏引导，点击开始）         │
│    ↓                                    │
│  分析页（实时进度 + 预览图）             │
│    ↓                                    │
│  报告页（核心结论 + 可展开详情）         │
└────────────────────────────────────────┘
→ 从5步减少到4步，减少页面跳转感
```

```
【问题2：表单割裂感】
原：先填年龄性别，再拍照，再等结果

优化：拍照优先，信息后置
┌────────────────────────────────────────┐
│  拍照 → AI分析（不等待填信息）          │
│    ↓                                    │
│  报告预览（提示"补充年龄获得更准建议"）  │
│    ↓                                    │
│  补充信息 → 获得个性化方案              │
└────────────────────────────────────────┘
```

**优化方案1：单页流 + 最小信息收集**
- 首页只放一个大按钮："开始舌诊"
- 拍照引导内嵌，点击"开始"直接调起相机
- 分析过程不离开当前页面
- 报告在本页滚动展示

**优化方案2：拍照优先架构**
- 第一优先级：完成舌象采集
- 第二优先级：AI即时分析（不阻塞信息填写）
- 第三优先级：用户补充信息（可跳过，不影响基础报告）

### 4.2 交互流程优化

**核心原则：AI优先，手动兜底**

```
【舌诊主流程优化】

Step 1: 拍照（0门槛）
┌────────────────────────────────────────┐
│  智能光线检测 + 构图引导                │
│  实时预览舌象是否合格                  │

│  • 体质初步判断                         │

│  前端显示：进度条 + 优秀设计展示        │
└────────────────────────────────────────┘
            ↓
Step 3: 报告展示（分层）
┌────────────────────────────────────────┐
│  第一眼可见（无需操作）：               │
│  • 核心体质类型（大字）                │
```

**优化方案3：智能默认 + 修正兜底**
- 年龄：不强制填写，默认"成人(18-65)"，显示"点击修改更精准"
- 性别：微信授权获取，未授权则显示"未设置"，不阻断流程
- 历史用户：自动填充上次信息，只需确认

**优化方案4：症状预填 + 用户确认**
- 参考看舌头APP模式：
  ```
  AI根据舌象推测："您可能有以下症状，请确认："
  ┌─────────────────────────────────┐
  │ □ 容易疲劳                      │
  │ □ 睡眠不好                      │
  │ ☑ 消化不良 ← 用户已确认        │
  │ □ 口干口渴                      │
  │   [都没有？点击跳过]             │
  └─────────────────────────────────┘
  ```
- 用户只需勾选"有/没有"，不用自己回忆

### 4.3 年龄/性别必填项处理

**方案对比**

| 方案 | 实现 | 优点 | 缺点 | 推荐度 |
|------|------|------|------|--------|
| A. 完全不问 | 只靠舌象辨体质 | 最简单 | 可能不准 | ⭐⭐⭐ |
| B. 弱提示收集 | 报告页提示"补充年龄更准"，可点可不点 | 无压力 | 转化率低 | ⭐⭐⭐⭐ |
| C. 智能推断 | 从微信昵称/头像AI推断年龄区间 | 无感 | 不保证准 | ⭐⭐⭐ |
| D. 默认+修改 | 年龄默认"30岁"，性别默认"未设置" | 简单 | 可能不准 | ⭐⭐⭐⭐⭐ |
| E. 后置收集 | 先看报告，底部"完善信息获取专属方案" | 不阻断 | 可能不看 | ⭐⭐⭐⭐ |

**推荐策略组合**：
```
【首次用户】B+C混合
  → 报告页顶部提示"完善信息获得个性化建议"
  → 同时AI尝试从昵称推断
  → 用户点"完善"再填，不点直接看报告

【回访用户】D+E混合
  → 自动填充历史信息
  → 显示"上次您是35岁男性，点击修改"
  → 一键确认比修改更方便
```

## 六、总结

### 核心结论

1. **"人性本懒"是产品设计的核心杠杆**
   - 能AI自动完成的，绝不让用户手动做
   - 必须用户参与的，给最小阻力
   - 保留手动修正能力，满足控制欲

2. **极简流程是民用产品的制胜关键**
