# 舌镜验收标准（ACCEPTANCE.md）

> 定义舌镜产品各环节的交付标准，让每次改动"有据可依"。
> 原则：准→快→稳，验收标准是"稳"的基石。

---

## 一、AI舌象识别验收

### 1.1 识别字段完整率

| 字段 | 必须识别 | 可选 | 说明 |
|------|---------|------|------|
| 舌色 | ✅ | | 淡红/淡白/红/绛/紫/青紫，至少命中一个 |
| 舌形 | ✅ | | 胖大/瘦薄/正常 |
| 舌态 | | ✅ | 强硬/痿软/歪斜/颤动/正常 |
| 苔色 | ✅ | | 薄白/白厚/黄/灰黑/剥落 |
| 苔质 | | ✅ | 薄/厚/正常 |
| 润燥 | | ✅ | 润/燥/正常 |
| 腐腻 | | ✅ | 腻/腐/否 |
| 齿痕 | | ✅ | 是/否 + 程度 + 分布 |
| 裂纹 | | ✅ | 是/否 + 程度 + 分布 |

**通过标准**：
- 🟢 必须字段命中率 ≥ 95%（100张测试图中 ≥ 95张舌色+舌形+苔色三字段均有输出）
- 🟡 必须字段命中率 80-95%，需排查原因
- 🔴 必须字段命中率 < 80%，不可上线

### 1.2 识别结果无崩溃

- 识别返回数组不得含 undefined、null、非字符串值
- 所有字段遍历前必须做空值守卫（.filter(Boolean)）
- **P0红线**：任何识别结果不得导致页面白屏或JS报错

### 1.3 识别→特征区联动

- AI识别结果必须自动填充到特征区（mapRecognitionToInputFeatures）
- 填充前先清空旧数据，没识别到的字段留空
- 手动修改特征后，智能识别区同步更新
- 纯手动模式（不上传图片）也能正常辨证

## 二、辨证引擎验收

### 2.1 四层推理完整性

| 层级 | 职责 | 验收点 |
|------|------|--------|
| Layer1 | 舌质舌苔→气血脾胃判断 | 必须输出 tongueBodyConclusion + coatingConclusion + compoundConclusion |
| Layer2 | 舌形舌态→脏腑定位 | 必须输出脏腑定位结论 + 置信度 |
| Layer3 | 区域诊断→病位细化 | 舌尖/舌边/舌中/舌根分区推理完整 |
| Layer4 | 综合辨证→证型+方案 | 必须输出主证型 + 证型得分 + 病机 + 针灸方案 |

**通过标准**：
- 四层全部执行，无跳层、无空层
- 每层都有 reasoningChain（推理链可追溯）
- 主证型 confidence ≥ 0.5 才可展示，< 0.5 标记"仅供参考"

### 2.2 辨证结果质量

- **主证型**不得为空，不得为通用词（如"适度调理"需附带具体证型）
- **病机**不得为空，必须是一句完整的病机描述
- **主穴**数量 ≥ 1，不得为 0
- **次要证型**最多展示 3 个，按 score 降序
- **治疗建议**区域不得为空（治疗原则+主穴+配穴+生活调护四项齐全）

### 2.3 正常舌象处理

- 正常舌象（淡红舌薄白苔）辨证结果应明确标注"正常/平和体质"
- 不得出现"适度调理"等模糊表述而不给出具体证型
- 主穴可为 0，但必须给出"日常养生建议"替代

## 三、智能问诊验收

### 3.1 问诊生成

- 问诊问题必须基于中医十问歌体系，不得自由发挥
- 根据初步辨证选 3-5 个相关追问
- 问题不得重复，不得与已选症状重叠

### 3.2 问诊后辨证

- 问诊得到的辨证内容必须用"（问诊）"标注区分
- 综合辨证方案中，望诊+问诊结论合并展示，结构清晰
- 问诊提交后不得出现辨证内容变空

### 3.3 问诊流程稳定性

- 点击"智能问诊"按钮后不得卡在"分析中"超 30 秒
- 问诊对话框打开/关闭不得导致页面状态丢失

## 四、性能验收

| 指标 | 标准 | 测量方式 |
|------|------|----------|
| AI识别响应 | ≤ 15秒 | 从上传图片到识别结果返回 |
| 望诊辨证 | ≤ 10秒 | 从点击辨证到结果展示 |
| 问诊生成 | ≤ 10秒 | 从触发问诊到问题展示 |
| 页面首屏 | ≤ 3秒 | FCP（First Contentful Paint） |
| 识别→特征填充 | ≤ 0.5秒 | 识别结果到特征区渲染完成 |

---

## 五、数据流验收

### 5.1 核心数据链路

```
AI识别 ──→ 特征区(自动填充) ──→ 智能识别区 ──→ 望诊辨证
手动选择 ──→ 特征区 ──────────→ 智能识别区 ──→ 望诊辨证
```

**验收点**：
- 识别→特征区：自动填充，先清空再填充
- 特征区→智能识别区：实时同步，只读展示
- 手动修改特征后智能识别区立即更新
- 辨证提交时以智能识别区的数据为最终输入

### 5.2 数据不丢失

- 页面刷新后患者信息+特征选择+辨证结果持久化（zustand persist）
- 问诊对话内容在关闭对话框后不丢失
- 病例记录可回溯

## 七、回归检查清单

每次代码变更上线前，必须逐项确认：

- [ ] AI识别→特征区联动正常
- [ ] 手动选择特征后辨证正常
- [ ] 辨证结果四层完整，无空字段
- [ ] 问诊流程不卡住，辨证内容不丢失
- [ ] 正常舌象辨证有明确结果
- [ ] 页面无JS报错，无白屏
- [ ] npm run build 通过
